00070 // Use of (void*) to store opaque data is now discouraged in order to properly
00071 // support 64-bit platforms. For backward compatibility with existing client
00072 // software, we can still provide (void*) API in addition to the new (intptr_t)
00073 // API. Ideally, clients should convert away from (void*) usage, but doing so
00074 // may be a large task, so we enable support for the deprecated API for now.
00075 // In order to aid clients in the conversion, we provide two controls. If
00076 // AWS_OBSOLETE_VOIDP is defined by the client, then all methods dealing with
00077 // (void*) will be removed completely from the API. If AWS_DEPRECATE_VOIDP is
00078 // defined, then invocations of methods dealing with (void*) will emit warnings
00079 // (from compilers which support deprecation warnings).
00080 #ifdef AWS_OBSOLETE_VOIDP
00081 #define AWS_VOIDP_IS_ERROR
00082 #endif
00083 #ifdef AWS_DEPRECATE_VOIDP
00084 #define AWS_VOIDP_IS_WARNING CS_DEPRECATED_METHOD
00085 #else
00086 #define AWS_VOIDP_IS_WARNING
00087 #endif
