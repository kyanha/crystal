Construct an input description from a string.

Parameters:
name_reg A pointer to the event name registry.
string The string to parse, e.g. "mousebutton1", "shift+a".
honorModifiers A bitmask of modifier keys that will be recognised.
useCookedCode If true, will use the cooked key code instead of raw. More precisely, the syntax for string is (in EBNF): 
 [ mod ("+"|"-") ]* ( ( nonZeroDevNumber? ("mouse" | "joystick") ( "X" | "Y" | ("Axis" axisNumber) | ("Button" buttonNumber) | buttonNumber ) ) | key )
So the fourth axis of the third joystick is "2JoystickAxis3", and its first button is "2JoystickButton0" or "2Joystick0". The leading number must be omitted for the first (primary) input device, e.g., "MouseAxis0" or "JoystickButton3".
