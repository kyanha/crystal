00044 /* The 'class csColliderWrapper' is here to work around a VC7.0 issue. It 
00045  * seems that, depending on whether VC7.0 sees a compound declared as
00046  * 'struct' or 'class' first, it sometimes generated different decorated
00047  * symbol names. 'csColliderWrapper' is a class, so commonly, in headers
00048  * etc. it would be forward-declared as 'class csColliderWrapper'. However,
00049  * the SCF_VERSION causes a forward declaration as a 'struct', with the
00050  * consequence that when csColliderWrapper was not already forward-declared
00051  * all decorated name will refer to 'struct csColliderWrapper' in some files.
00052  * But possibly, when forward-declaration was used, 'class csColliderWrapper'
00053  * will be used in other objects, causing linking errors. This is worked
00054  * around by having a 'class' forward declaration here.
00055  */
00056 class csColliderWrapper;
00057 SCF_VERSION (csColliderWrapper, 0, 0, 3);
