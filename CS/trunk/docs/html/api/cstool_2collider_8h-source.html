00046 /* The 'class csColliderWrapper' is here to work around a VC7.0 issue. It 
00047  * seems that, depending on whether VC7.0 sees a compound declared as
00048  * 'struct' or 'class' first, it sometimes generated different decorated
00049  * symbol names. 'csColliderWrapper' is a class, so commonly, in headers
00050  * etc. it would be forward-declared as 'class csColliderWrapper'. However,
00051  * the SCF_VERSION causes a forward declaration as a 'struct', with the
00052  * consequence that when csColliderWrapper was not already forward-declared
00053  * all decorated name will refer to 'struct csColliderWrapper' in some files.
00054  * But possibly, when forward-declaration was used, 'class csColliderWrapper'
00055  * will be used in other objects, causing linking errors. This is worked
00056  * around by having a 'class' forward declaration here.
00057  */
00058 class csColliderWrapper;
00059 SCF_VERSION (csColliderWrapper, 0, 0, 3);
