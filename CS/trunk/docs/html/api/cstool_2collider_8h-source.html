00051 /* The 'class csColliderWrapper' is here to work around a VC7.0 issue. It 
00052  * seems that, depending on whether VC7.0 sees a compound declared as
00053  * 'struct' or 'class' first, it sometimes generated different decorated
00054  * symbol names. 'csColliderWrapper' is a class, so commonly, in headers
00055  * etc. it would be forward-declared as 'class csColliderWrapper'. However,
00056  * the SCF_VERSION causes a forward declaration as a 'struct', with the
00057  * consequence that when csColliderWrapper was not already forward-declared
00058  * all decorated name will refer to 'struct csColliderWrapper' in some files.
00059  * But possibly, when forward-declaration was used, 'class csColliderWrapper'
00060  * will be used in other objects, causing linking errors. This is worked
00061  * around by having a 'class' forward declaration here.
00062  */
00063 class csColliderWrapper;
00064 SCF_VERSION (csColliderWrapper, 0, 0, 3);
