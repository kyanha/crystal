2.5.1 MacOS/X

Mac OS X Lion 10.7

‘generic thunk’ compiling error

Newer XCode (versions 4.2 and later) come with the default compiler llvm-gcc-4.2 (GCC) 4.2.1 (Based on Apple Inc. build 5658) which has problems handling variable arguments lists in the presence of multiple-inheritance, resulting in a failure to compile CS by returning a ‘generic thunk’ error. One way of solving this is by installing a previous version of XCode (version 4.1 for instance) that installs gcc (GCC) 4.2.1 (Apple Inc. build 5666) (dot 3) which should compile CS without errors. The default compiler may be set by making the gcc / g++ / cc / c++ commands link to the compiler we want:

 
which gcc -> /usr/bin/gcc
ln -s -f /usr/bin/i686-apple-darwin11-gcc-4.2.1 /usr/bin/gcc

Undefined symbols for architecture i386: "_main"

Gcc version 4.2.1 mentioned above can require a main() dummy function for plugins as is the case with Unix. For this a similar method to CS_IMPLEMENT_PLATFORM_PLUGIN from CS/include/csutil/unix/csosdef.h should be added to CS/include/csutil/macosx/csosdef.h.

 
#define CS_IMPLEMENT_PLATFORM_PLUGIN \
int main (int argc, char* argv[]) \
{ \
  (void)argc; (void)argv; \
  return 0; \
}

DLERROR: can't map unslidable segment

This is likely to be caused by the fact that the generated .csbundle files are executables instead of bundles. To check for this run the file shell command on a .csbundle file:

 
file vfs.csbundle -> vfs.csbundle: Mach-O executable i386

If this is the case double check if the ‘-bundle’ flag (PLUGIN.LFLAGS) is present for the mac jam configuration (CS/mk/jam/macosx.jam).

 
file vfs.csbundle -> vfs.csbundle: Mach-O bundle i386
