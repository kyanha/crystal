4.18 Alternate Windowing System (AWS)

Written by Christopher Nelson, paradox@bbhc.org. Original AWS documentation converted to Texinfo by F. Richter, res@crystalspace3d.org.

The Alternate Windowing System is a plugin designed to replace the old monolithic (and retired) Crystal Space Windowing System (CSWS). It utilizes an region optimal frame-thrower and can draw to either the default context, or to any user-supplied context (such as a procedural texture). AWS also supports a definition system and access to components via a signal/slot mechanism. All GUI aspects can be modified by the GUI designer in definition files without the need to recompile the base application. This allows game interfaces to be easily skinned by non-programmers. For example code, see `CS/apps/tutorial/awstut' and `CS/apps/tests/awstest'.
