00041 /* The 'class csColliderWrapper' is here to work around a VC7.0 issue. It 
00042  * seems that, depending on whether VC7.0 sees a compound declared as
00043  * 'struct' or 'class' first, it sometimes generated different decorated
00044  * symbol names. 'csColliderWrapper' is a class, so commonly, in headers
00045  * etc. it would be forward-declared as 'class csColliderWrapper'. However,
00046  * the SCF_VERSION causes a forward declaration as a 'struct', with the
00047  * consequence that when csColliderWrapper was not already forward-declared
00048  * all decorated name will refer to 'struct csColliderWrapper' in some files.
00049  * But possibly, when forward-declaration was used, 'class csColliderWrapper'
00050  * will be used in other objects, causing linking errors. This is worked
00051  * around by having a 'class' forward declaration here.
00052  */
00053 class csColliderWrapper;
00054 SCF_VERSION (csColliderWrapper, 0, 0, 3);
