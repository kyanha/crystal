00071 // Use of (void*) to store opaque data is now discouraged in order to properly
00072 // support 64-bit platforms. For backward compatibility with existing client
00073 // software, we can still provide (void*) API in addition to the new (intptr_t)
00074 // API. Ideally, clients should convert away from (void*) usage, but doing so
00075 // may be a large task, so we enable support for the deprecated API for now.
00076 // In order to aid clients in the conversion, we provide two controls. If
00077 // AWS_OBSOLETE_VOIDP is defined by the client, then all methods dealing with
00078 // (void*) will be removed completely from the API. If AWS_DEPRECATE_VOIDP is
00079 // defined, then invocations of methods dealing with (void*) will emit warnings
00080 // (from compilers which support deprecation warnings).
00081 #ifdef AWS_OBSOLETE_VOIDP
00082 #define AWS_VOIDP_IS_ERROR
00083 #endif
00084 #ifdef AWS_DEPRECATE_VOIDP
00085 #define AWS_VOIDP_IS_WARNING CS_DEPRECATED_METHOD
00086 #else
00087 #define AWS_VOIDP_IS_WARNING
00088 #endif
