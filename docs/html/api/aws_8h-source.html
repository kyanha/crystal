00069 // Use of (void*) to store opaque data is now discouraged in order to properly
00070 // support 64-bit platforms. For backward compatibility with existing client
00071 // software, we can still provide (void*) API in addition to the new (intptr_t)
00072 // API. Ideally, clients should convert away from (void*) usage, but doing so
00073 // may be a large task, so we enable support for the deprecated API for now.
00074 // In order to aid clients in the conversion, we provide two controls. If
00075 // AWS_OBSOLETE_VOIDP is defined by the client, then all methods dealing with
00076 // (void*) will be removed completely from the API. If AWS_DEPRECATE_VOIDP is
00077 // defined, then invocations of methods dealing with (void*) will emit warnings
00078 // (from compilers which support deprecation warnings).
00079 #ifdef AWS_OBSOLETE_VOIDP
00080 #define AWS_VOIDP_IS_ERROR
00081 #endif
00082 #ifdef AWS_DEPRECATE_VOIDP
00083 #define AWS_VOIDP_IS_WARNING CS_DEPRECATED_METHOD
00084 #else
00085 #define AWS_VOIDP_IS_WARNING
00086 #endif
